#AI API POOL
如今提供模型api的厂商千千万，他们都有免费的试用活动可以白嫖福利，同时我们有时候也要对我们的ai后端做一些容灾冗余，所以我们需要一个代理池

## 细节
* 比如，openrouter，阿里云，azure,deepseek都提供了deepseek-r1的api,他们大多使用openai兼容的api，那么我们需要创建一个代理池，将这些api集成起来，本地开一个server，接收调用端的请求，将这些请求负载均衡到这些api上，同时要有对api的可用性的检测，当某一个api返回余额不足或者下线时，本次运行中就不再选择这个api。
* 用rust写
* 应当分前后端，后端负责api的集成与转发，前端就是一个小webserver，可以对外暴露本地api，这么做是方便在其他代码中引用，或者不集成到项目中单独开服务器。
* 配置文件用yaml
* 可能不止代理一个模型，不同模型应当独立配置，互相隔离
  - 这一点类似于其他的模型api提供商
